Sistema de reportes automatizados

Descripción del proyecto

Este proyecto consiste en un sistema ETL (Extracción, Transformación y Carga) desarrollado en Python utilizando la biblioteca Tkinter para la creación de una interfaz gráfica intuitiva. El objetivo principal del sistema es optimizar el proceso de generación de reportes diarios para diferentes áreas de la empresa, facilitando la carga y manipulación de datos desde un servidor SQL.

Funcionalidades principales

  1. Interfaz grafica:

    El sistema presenta una interfaz amigable donde cada área de la empresa cuenta con un botón dedicado. La organización de las funciones se realiza mediante pestañas, utilizando el widget Notebook de Tkinter, lo que permite un acceso rápido y fácil a las diferentes campañas, incluyendo una específica para la campaña de delivery.

  2. Proceso ETL:
    • Carga de Datos: Se proporciona un apartado para cargar los datos desde el servidor SQL.
    • Ejecución de Procedimientos: Los usuarios pueden ejecutar procedimientos almacenados directamente desde la interfaz, simplificando el proceso.
  3. Exportación y Reportes:

    Una de las características clave del sistema es la capacidad de exportar datos a archivos Excel. Los usuarios pueden seleccionar la fecha deseada para extraer los reportes, los cuales se almacenan en un archivo Excel que se actualiza automáticamente con los datos correspondientes.

  4. Validación de Errores y Mensajes Informativos:

    El sistema incluye validaciones para detectar errores durante el proceso de carga y ejecución, asegurando una mayor confiabilidad. Además, se implementan mensajes informativos mediante messagebox, que avisan a los usuarios sobre el estado de la ejecución, incluyendo una barra de progreso que muestra el avance del procedimiento.

  5. Seguridad:

    Se ha implementado un sistema de login que garantiza el acceso seguro a la aplicación, permitiendo gestionar los permisos de los usuarios de manera efectiva.

Tecnologías utilizadas

  • Python: Lenguaje de programación principal utilizado para desarrollar el script.
  • Tkinter: Biblioteca para la interfaz gráfica.
  • SQLAlchemy: Conexión con la base de datos.
  • PyInstaller: Empaquetado de la aplicación.

Obejtivos

El sistema tiene como objetivo principal reducir el tiempo de ejecución y carga de datos para la generación de reportes diarios, mejorando así la eficiencia operativa de la empresa. Con esta herramienta, se busca minimizar errores y optimizar el flujo de trabajo, permitiendo que los equipos se concentren en tareas de mayor valor añadido.

Conclusiones

Este sistema ETL representa una solución innovadora para las necesidades de reportería de la empresa, integrando tecnología moderna con una interfaz accesible. Con su implementación, se espera un notable aumento en la productividad y una disminución en el tiempo dedicado a la gestión de datos.
